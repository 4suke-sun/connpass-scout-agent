# 仕様不足時の確認スキル

## 使用タイミング
以下のいずれかに該当する場合:
- リクエストに受け入れ条件がない
- 複数の実装が同等に妥当
- スコープが不明確（どのファイル？どの環境？）
- セキュリティやデータへの影響が言及されていない
- リクエストが既存の規約と矛盾している

## 手順

1. 最も重要な未確認事項を 1〜5 個特定する（網羅的でなくてよい — ブロッカーに集中する）。
2. 番号付きの質問として整形する。
3. 回答を待つまで作業を開始しない。

## 質問テンプレート

```
作業を始める前に、いくつか確認させてください:

1. **[トピック]**: <具体的な質問>？
2. **[トピック]**: <具体的な質問>？
3. **[トピック]**: <具体的な質問>？

（まとめて回答しても、1つずつでも構いません。）
```

## ルール

- 1回あたり最大5問まで。
- コードベースを検索すれば分かる質問はしない。
- Biome/tsconfig で既に決まっているスタイルの好みについて質問しない。
